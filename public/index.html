Stellar MPP Demo

Why should builders care about MPP?

MPP turns payment into a protocol-level capability: apps and agents can discover paid resources, receive a standard challenge, pay on Stellar, and continue the same API or MCP tool call.

So what?

APIs can sell tiny units of value

A server can say “this response costs 0.00001 XLM” without building accounts, invoices, API-key billing, or a separate checkout flow.

For agents

Tools become directly monetizable

MCP tools can be free for lookup and paid for deeper analysis. The payment challenge travels inside the tool call metadata.

For scale

Channels make repeated calls practical

Charge mode is great for one request. Channel mode lets agents keep using paid tools through vouchers without a new on-chain tx per call.

Protocol notes

  • Charge pull: client signs, server submits, optional fee payer can sponsor fees.
  • Charge push: client signs and submits, then proves payment with a tx hash.
  • Channel mode: deposit and close are on-chain; each request signs a cumulative voucher verified through a read-only Soroban contract call.
  • Stellar Explorer MCP uses free lookup tools and paid analysis tools with payment challenges inside MCP metadata.
  • Secrets stay in the local server. The browser controls demo wallets and actions.
